It was designed with the idea of analyzing the differences between the monomers of PNP along the binding path of the ligand but its use can be extended in the folowing way. Use as input a file with N columns that can be divided into N/3 equivalent parts or monomers A, B, and C. The divisions are such that column i in monomer A is equivalent to columns i+ N/3 and i+2N/3 in monomers B and C respectively. This tool (1) uses kernel density estimation (KDE) to obtain the probability density distribution of each column, (2) analyses the differences between the KDE probability distributions of each set of equivalent columns; that is, for equivalent columns Aj, Bj, and Cj we calculate the distances d(Aj,Bj), d(Aj,Cj), and d(Bj,Cj). (3) next we determine the 'excess' in which a distance dominates the others. In practice, to see how far is Aj from the others we calculate Q_Aj = mean([d(Aj,Bj), d(Aj,Cj)]) - mean([d(Aj,Bj), d(Aj,Cj), d(Bj,Cj) ]), if mean([d(Aj,Bj), d(Aj,Cj)]) > mean([d(Aj,Bj), d(Aj,Cj), d(Bj,Cj) ])  (4) we also sum all Q_Aj, Q_Bj, and Q_Cj to have an estimate of the total excess of monomers A, B and C, respectively. The results are shown by column and by monomer for all the provided data files or images.",epilog="Thanks for using trimerKDE.py!!")
